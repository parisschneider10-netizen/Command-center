# Budget Law — Least Money, Never Cheap Quality

> **Paramount:** Minimum spend and Commander energy. System quality stays sovereign-grade. Overruns require nuclear approval.

---

## Principle

| Cheap operations | Never cheap |
|------------------|-------------|
| One VPS until revenue | Code, doctrine, security |
| Dry-run drills | Health checks, HTTPS |
| Presale-funded closers | SARA verified-truth protocol |
| sslip.io vs domain (early) | Treasury ledger audit trail |
| A2A before humans | Human firewall for physical only |

## Automatic overrun checks (in code)

These run **without you watching**:

```
Spend request
  → under guardian cap ($25)? → auto-approve
  → over cap?                 → nuclear queue (needs_commander)
  → over daily wallet limit?  → hold

Expansion scale
  → EXPANSION_DRY_RUN=true?   → simulate only
  → live batch > cap?         → reject until funded

Intent execute
  → RentAHuman micro-tasks    → budget from template ($20–45)
  → auto_post_rah             → only within caps

A2A hire
  → A2A_DAILY_CAP_USD ($100) → daily runway
```

## Spend order (cheapest path to revenue)

1. **$0** — Voice intent + GitHub `@cursor` builds  
2. **$20/mo** — Servury VPS (already live)  
3. **$0 until presale** — Closer bounties paid from **host $150**, not you  
4. **Float compounds** — $120 net → next city; $20 Cursor earmark from play  
5. **Voice exit** — LiveKit when treasury `voice` line clears (drop Vapi rent)  

---

## Commander anti-overrun habits

1. Always say **drill** or **dry run** before **launch** / **live** on a new play.  
2. Never flip `EXPANSION_DRY_RUN=false` until one presale succeeds.  
3. Never add API keys until the play needs them that day.  
4. Parallel cities only when treasury human-capital briefing says float supports it.  
5. Portal is for **nuclear** — not daily ops.
